ENTITY FRAMEWORK

Faz a ponte entre os dados e codigo
dentro do projeto cria um contexto,
atraves do framework ele cria uma replica das tabelas no codigo
o contexto
cria um diagrama de classes com base nas tabelas do bd dps transforma em classes
pu seja cria o bd primeiro, dps so pede pra ele transformar

existe um abismo entre sql e codigo
pode parecer serem iguais e estarem proximos,
mas a diferença é gigantesca

sempre que mexemos com determinada classe, como cliente
estamos mexendo literalmente com a classe
dps cabe ao codigo unir as mudanças do codigo para o banco


*Expressões Lambdas*

usadas por nos inicialmente em listas, where, select
so trazer/configurar para o contexto, para o entity
